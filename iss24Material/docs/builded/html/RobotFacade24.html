RobotFacade24¶

progetto RobotFacade24 (RbtFcd24)

RbtFcd24: il progetto¶

Lo scopo di questo lavoro è usare SpringBoot -> per costruire una applicazione Web (RbtFcd24) che fornisca una console di comando per il BasicRobot24.

Il progetto robotfacade24 è una evoluzione di Facade24Start.

Il lettore è invitato a

  1. illustrare come ritiene che siano aggiornate le varie parti della pagina con riferimento a quanto già introddto in La parte dinamica e in Fcd24S: interazioni.

  2. prefigurare il modo con cui sono state impostate le azioni relative ai pulsanti envmap e robotpos

  3. prefigurare il modo in cui i dati di WEnv relativi al sonar vengono riportati nella sezione Sonar lastval

RbtFcd24: prima attivazione ed uso¶

1) Attivazione del virtualrobot¶

  1. Posizionarsi sulla directory unibo.basicrobot24

  2. Attivare il VirtualRobot con il comando docker-compose -f virtualRobot23.yaml up

2) Attivazione del basicrobot24¶

Premesse:

  1. verificare che nel modello basicrobotqak sia commentata la specifica

Facade robotfacade ...

  1. verificare che il file basicrobotConfig.json abbia il seguente contenuto:

{"type":"virtual", "port":"8090", "iprobot":"localhost", "commtrace": "false"}

Successivamente:

  1. Posizionarsi sulla directory unibo.basicrobot24

  2. Attivare il basic robot con il comando gradlew run

3) Attivazione della Facade¶

  1. Posizionarsi sulla directory robotfacade24

  2. Attivare la Facade con il comando gradlew run

  3. Usare la Facade mediante i pulsanti

Attivazione intergrata della Facade¶

Premessa: introdurre nel modello basicrobotqak la specifica

Facade robotfacade port 8085 usingactor basicrobot inctx ctxbasicrobot
               appl "main.java.robotfacade24.Robotfacade24Application"
               UserLibs -f "robotfacade24-1.0-plain"  -f "robotfacade24-1.0"

Assicurarsi di avere copiato le librerie robotfacade24-1.0-plain e robotfacade24-1.0 che vengono automaticamente refenziate nel file build2024.gradle (sezione USER libs for facades) del progetto basicrobot24.

RbtFcd24: una applicazione¶

progetto boundaryBasicrobot24

Realizzare i primi 2 requisiti del progetto BoundaryWalk24 usando il BasicRobot24 e osservare le informazioni sulla FacadeGui

RbtFcd24: creare una immagine Docker¶

Premessa:

  1. verificare che il file basicrobotConfig.json abbia il seguente contenuto:

    {"type":"virtual", "port":"8090", "iprobot":"IPCOMPUTER", "commtrace": "false"}

  2. Consultare ed eseguire quanto riportato nel file Dockerfile del progetto.

  3. Eseguire l’immagine

    docker run -it --rm --name basicrobot24 -p8020:8020/tcp -p8020:8020/udp -p8085:8085/tcp --privileged basicrobot24:1.0

RbtFcd24: flie yaml¶

  1. verificare che il file basicrobotConfig.json abbia il seguente contenuto:

    {"type":"virtual", "port":"8090", "iprobot":"wenv", "commtrace": "false"}

  2. Creare l’immagine come riportato in RbtFcd24: creare una immagine Docker

  3. Consultare il file yaml: webbasicrobot24.yaml

  4. Eseguire

    docker run -it --rm --name basicrobot24 -p8020:8020/tcp -p8020:8020/udp -p8085:8085/tcp --privileged basicrobot24:1.0
