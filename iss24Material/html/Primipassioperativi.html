Primi passi operativi¶

Installazioni su PC¶

  1. Installare Java JDK 11 (non useremo versioni più recenti)

  2. Installare GIT PERSONALE PRIVATO e dare accesso al docente

  3. Compilare template2023 inserendo Fotografia, email e numero di matricola

  4. Clonare https://github.com/anatali/issLab23 in una directory vuota e.g. C:/.../issLab23

  5. Installare Gradle (7.6)

  6. Installare IntelliJ

  7. Installare Eclipse IDE for Java and DSL Developers (2022 06)

  8. Installare Docker

  9. Installare Node.js

  10. Installare Python (3.8.8)

Creazione progetto con Gradle¶

Gradle è uno strumento open source per automatizzare la costruzione (build) del software.

Gradle è ormai uno standard di fatto in questo settore ed è interessante non solo in quanto strumento, ma anche perchè applica i principi del Domain Driven Design (DDD) per modellare il suo proprio domain-building software.

Per un approfondimento su Gradle si veda GradleIntro.

Iniziamo creando un progetto con l’aiuto di Gradle

  1. Apro sul PC una directory di lavoro (ad esempio issLab23) e mi posiziono in essa.

  2. Creo la directory unibolibs, che conterrà le librerie (file .jar) da noi sviluppate. Per il momento, inserisco le seguenti librerie:

    • unibo.basicomm23-1.0

  3. Creo una directory per il progetto di nome Appl1 e mi posiziono in essa.

  4. Eseguo il comando

    gradle init
(select 2, 3, 1, 1, -, 1, -, -)

    La risposta 2 alla prima domanda, implica che Gradle costruisca una applicazione-base che possiamo subito eseguire e testare.

  5. Eseguo il codice generato:

    gradlew build -x test  //esclude i test
gradlew run

    L’output dovrebbe essere:

    > Task :app:run
Hello World!
BUILD SUCCESSFUL in 2s
2 actionable tasks: 1 executed, 1 up-to-date

  6. Apro il progetto in IntelliJ , noto le directories main e test

  7. Osservo il contenuto di app/build.gradle:

    plugins {
    // Apply the application plugin
    //to add support for building a CLI application in Java.
    id 'application'
}
repositories {
    // Use Maven Central for resolving dependencies.
    mavenCentral()
}
dependencies {
    // Use JUnit test framework.
    testImplementation 'junit:junit:4.13.2'
    // This dependency is used by the application.
    implementation 'com.google.guava:guava:30.1.1-jre'
}
application {
    // Define the main class for the application.
    mainClass = 'Appl1.App'
}

    L’applicazione del plug-in application applica implicitamente anche il plug-in Java, il quale assume src/main/java e src/test/java come directory di origine predefinite.

  8. Osservo il contenuto del file .gitignore:

    # Ignore Gradle project-specific cache directory
.gradle
# Ignore Gradle build output directory
build

  9. Eseguo i test (usando una command shell o il Terminal di IntelliJ):

    gradlew build   //does also the testing
gradlew test    //does only the tests

  10. Osservo app/build/reports/tests/test/index.html

Lasciamo come pro-memoria questa configurazione e creaimo la nostra applicazione

Applicazione¶

  1. Creo una nuova directory application1 entro Appl1

  2. Nel file settings.gradle modifico la frase include(‘app’) in include('application1')

  3. Creo in application1 le directories (come in app):

    • src/main/java e src/main/resources

    • src/test/java e src/test/resources

  4. Marco src come source directory

  5. Creo la directory src/main/java/unibo, che funge da nuovo package

  6. Creo in application1/src/main/java/unibo un file di prova Application1.java

    package unibo;
import unibo.basicomm23.utils.CommUtils;
public class Application1 {
        public static void main( String[] args ){
            System.out.println("Application1 hello");
            CommUtils.outblue("Application1 hello again");
        }
}

  7. Creo in application1 un file build.gradle con il seguente contenuto:

    plugins {
   id 'application'
   id 'eclipse'
   id 'jacoco'
}

version '1.0'
sourceCompatibility = 1.8

repositories {
    mavenCentral()
    flatDir { dirs '../../unibolibs' }
}

dependencies {
    // Use JUnit test framework.
    testImplementation 'junit:junit:4.13.2'

    // This dependency is used by the application.
    implementation 'com.google.guava:guava:30.1.1-jre'
    //HTTP
    implementation 'org.apache.httpcomponents:httpclient:4.5'
    //JSON
    implementation 'com.googlecode.json-simple:json-simple:1.1.1'

    //CUSTOM unibo
    implementation name: 'unibo.basicomm23-1.0'
}

 application {
    // Define the main class for the application.
    mainClass = 'unibo.Application1'
}

  8. Usando una command shell o il Terminal di IntelliJ su C:\Didattica2023\issLab23\Appl1, eseguo:

    gradlew build
gradlew run

  9. Creo la directory Appl1/userDocs e vi inserisco in userDocs copia del file template2023 con la mia foto, lo ridenomino CognomeNomeApplication1.html e vi copio i requisiti dati dal committente.

Uso di GIT¶

Una volta creato il progetto, è opportuno salvarlo su un nostro repository GIT.

Per un aiuto ad usare GIT può essere utile consultare Basic Git commands e/o guardare il video Video on GIT di cui riportiamo l’inizio di alcuni punti salienti:

Per quanto riguarda il nostro progetto:

  1. Mi posiziono sulla directory issLab23.

  2. Eseguo:

    git init  //creates the directory .git
git status

  3. Osservo il contenuto del file generato .gitignore :

    # Ignore Gradle project-specific cache directory
.gradle
# Ignore Gradle build output directory
build

git status --ignored      //see ignored files

    I files elencati non saranno salvati sul repository.

  4. Eseguo i comandi .. code:

    git add -A
git commit -m "Appl1"
git log   //q to exit
git status

Creazione di un repository remoto¶

  1. Supponendo di avere accesso su github come user di nome userxyz, creiamo un repository personale di nome issLab23, selezionando il tipo private, con README file e Add .gitignore (template Java). Quindi aggiungiamo il nostro progetto al repository:

    git remote add origin https://github.com/userxyz/issLab23
git remote -v   //osservo

  2. Rendo visibile al docente del corso il progetto su github

  3. Posizionato sulla directory issLab23, salvo il progetto corrente sul repository remoto.

    git push origin master
